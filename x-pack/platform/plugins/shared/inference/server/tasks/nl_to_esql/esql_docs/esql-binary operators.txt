# BINARY OPERATORS

Binary operators are used to compare values or perform arithmetic operations between two fields or values. If either field is multivalued, the result is `null`.

## Syntax

`<field or value> <operator> <field or value>`

### Parameters

#### <operator>

The binary operator to use. Supported operators include:

- Equality: `==`
- Inequality: `!=`
- Less than: `<`
- Less than or equal to: `<=`
- Greater than: `>`
- Greater than or equal to: `>=`
- Add: `+`
- Subtract: `-`
- Multiply: `*`
- Divide: `/`
- Modulus: `%`

## Examples

Filters employees whose department is "Engineering".
```esql
FROM employees
| WHERE department == "Engineering"
```

Filters employees whose salary is not 50,000.
```esql
FROM employees
| WHERE salary != 50000
```

Filters employees younger than 30.
```esql
FROM employees
| WHERE age < 30
```

Filters employees with 5 or fewer years of experience.
```esql
FROM employees
| WHERE years_experience <= 5
```

Filters employees with a salary greater than 100,000.
```esql
FROM employees
| WHERE salary > 100000
```

Filters employees with a performance score of 90 or higher.
```esql
FROM employees
| WHERE performance_score >= 90
```

Calculates the total compensation by adding salary and bonus.
```esql
FROM employees
| EVAL total_compensation = salary + bonus
```

Calculates the years until retirement by subtracting the employee's age from 65.
```esql
FROM employees
| EVAL years_until_retirement = 65 - age
```

Calculates double the salary.
```esql
FROM employees
| EVAL double_salary = salary * 2
```

Calculates half the salary.
```esql
FROM employees
| EVAL half_salary = salary / 2
```

Calculates the remainder when salary is divided by 1,000.
```esql
FROM employees
| EVAL remainder = salary % 1000
```

## Limitations

- If either field is multivalued, the result is `null`.
- For comparison operators, the operation is pushed to the underlying search index if one side is a constant and the other is a field in the index with both a mapping-index and doc-values.
- Division of two integer types yields an integer result, rounding towards 0. For floating point division, cast one of the arguments to `DOUBLE`.
