# BINARY OPERATORS

Binary operators are used to compare values or perform arithmetic operations between two fields or values. If either field is multivalued, the result is `null`.

## Syntax

`<field_or_value> <operator> <field_or_value>`

### Parameters

#### <field_or_value>

A field or value to be compared or used in the arithmetic operation.

#### <operator>

The binary operator to apply. Supported operators include:

- Equality: `==`
- Inequality: `!=`
- Less than: `<`
- Less than or equal to: `<=`
- Greater than: `>`
- Greater than or equal to: `>=`
- Add: `+`
- Subtract: `-`
- Multiply: `*`
- Divide: `/`
- Modulus: `%`

## Examples

Filter employees whose department is "Engineering".

```esql
FROM employees
| WHERE department == "Engineering"
```

Filter employees whose status is not "inactive".

```esql
FROM employees
| WHERE status != "inactive"
```

Filter employees younger than 30.

```esql
FROM employees
| WHERE age < 30
```

Calculate the total by adding price and tax.

```esql
FROM sales
| EVAL total = price + tax
```

Calculate profit by subtracting cost from revenue.

```esql
FROM sales
| EVAL profit = revenue - cost
```

Calculate the total discount by multiplying quantity and discount.

```esql
FROM sales
| EVAL total_discount = quantity * discount
```

Calculate the average price by dividing revenue by quantity.

```esql
FROM sales
| EVAL avg_price = revenue / quantity
```

Calculate the remainder of 10 divided by 3.

```esql
ROW a = 10, b = 3
| EVAL remainder = a % b
```

## Limitations

- If either field is multivalued, the result is `null`.
- For comparison operators, the operation is pushed to the underlying search index if one side is a constant and the other is a field with both a mapping-index and doc-values.
- Division of two integer types yields an integer result, rounding towards 0. For floating point division, cast one of the arguments to `DOUBLE`.
